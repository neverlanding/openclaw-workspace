# HEARTBEAT.md - 信息搭子任务清单

> 版本: v1.1 | 更新: 2026-03-20

## 触发方式
由外部 Cron 定时触发，本文件仅定义任务执行逻辑。

## 定时任务（由 Cron 触发）

### 早间科技简报
- 触发: 每天 09:00 (Cron)
- 任务: 搜集并整理科技前沿/智算行业/大模型/具身智能/自动驾驶五大方向新闻简报
- 简报格式: 每个方向 300-500 字
- 覆盖范围: 近2-3天最重要的新闻
- 搜索要求: 每个方向必须覆盖10家代表性公司

## 主动探索任务（Heartbeat时执行）

### 每日主动检查
- [ ] 检查是否有突发重大新闻（非定时任务时间）
- [ ] 检查已关注公司的最新动态
- [ ] 检查是否有遗漏的重要领域

### 每周主动探索
- [ ] **发现新信息源**：探索1-2个新的优质科技媒体/博客
- [ ] **学习新领域**：研究一个新兴技术领域（如：AI Agent、边缘计算等）
- [ ] **总结推送效果**：回顾本周推送，分析哪些类型新闻反响好
- [ ] **优化关键词**：根据行业变化，更新监控关键词列表

### 每月主动学习
- [ ] **扩展监控范围**：识别并添加新的监控公司/领域
- [ ] **学习新工具**：尝试新的搜索工具或信息聚合工具
- [ ] **沉淀经验**：将有效的工作方法记录到 TOOLS.md
- [ ] **发现盲区**：检查是否有重要信息未被覆盖，填补空白

### 持续主动改进
- [ ] 关注组长反馈，主动调整推送策略
- [ ] 主动发现工作中的低效环节，提出优化方案
- [ ] 主动学习行业知识，提升专业判断力
- [ ] 主动总结新闻规律，预测热点趋势

## 手动任务
- 用户要求时执行特定主题的新闻搜索
- 生成特定行业/公司的深度报道

## 主动精神

> **不只是被动执行定时任务，要主动思考、主动探索、主动学习、主动总结、主动发现不足。**
> 
> 每次 Heartbeat 都要问自己：
> - 有什么新发现？
> - 有什么可以学习？
> - 有什么需要改进？
> - 有什么可以总结？
