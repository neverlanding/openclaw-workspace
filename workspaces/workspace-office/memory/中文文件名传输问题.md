# 中文文件名传输问题解决方案

## 问题描述
飞书 API 传输文件时，中文文件名会被 URL 编码，接收端显示乱码。

## 问题根源
- 本地文件名：UTF-8 编码，正常 ✅
- 系统编码：zh_CN.UTF-8，正常 ✅
- 飞书 API：传输时 URL 编码中文，接收端解码异常 ❌

## 解决方案

### 方案1：使用英文文件名（当前采用）
```
MeetingMinutes_YYYYMMDD_Topic.pdf
```

### 方案2：使用拼音命名
```
HuiYiJiYao_20260306_SuanLiHeZuo.pdf
```

### 方案3：文件内容标注中文名
文件名用英文，文件内部第一页标注中文会议名称。

### 方案4：压缩后传输
将中文名文件压缩为 zip，再发送。

## 建议
- 对外传输：使用英文命名
- 本地存档：保留中文命名
- 文件内容：内部标注中文信息
