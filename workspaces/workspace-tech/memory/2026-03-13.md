## 2. GitHub 配置备份与迁移系统

### 目标
建立完整的配置迁移方案，使新环境可以一键恢复当前 OpenClaw 配置

### 实现

#### 推送脚本
- 位置: `~/.openclaw/scripts/push-to-github.sh`
- 功能:
  - 收集 8 个 Agent 配置
  - 自动清理敏感信息（API Keys、Token）
  - 推送到 GitHub 仓库

#### 恢复脚本
- 位置: `~/.openclaw/workspace/github-config-backup-*/scripts/restore-from-github.sh`
- GitHub 地址: `https://github.com/neverlanding/openclaw-workspace/blob/master/scripts/restore-from-github.sh`
- 功能:
  - 安装 OpenClaw（如未安装）
  - 恢复所有 Agent 配置
  - 安装插件
  - 提示用户填入 API Keys

#### 使用方法
**新环境部署:**
```bash
git clone https://github.com/neverlanding/openclaw-workspace.git
cd openclaw-workspace
chmod +x scripts/restore-from-github.sh
./scripts/restore-from-github.sh
# 然后编辑 ~/.openclaw/openclaw.json 填入 API Keys
```

## 3. 迁移安全要点

### 敏感信息处理
- ✅ **GitHub 备份中已清理**: API Keys、Feishu appId/appSecret、Gateway Token
- ✅ **GitHub 备份中包含**: Agent 身份配置、头像、Skills、Memory

### 飞书配置限制
- **同一凭证不能多环境同时使用**
- 新环境必须手动配置飞书或创建新应用
- 原环境完全不受影响

### 两个环境共存方案
| 方案 | 可行性 | 操作 |
|:---|:---:|:---|
| 原环境停用，新环境使用 | ✅ 可行 | 原环境 `openclaw disconnect feishu` |
| 创建全新飞书应用 | ✅ 推荐 | 新应用在飞书开放平台创建 |
| 同时使用相同凭证 | ❌ 不可行 | 会导致冲突 |

## 4. 技术细节

### GitHub 认证方式
- 使用 Personal Access Token 认证
- Token 格式: `ghp_...`
- 需要 `repo` 权限

### 配置文件位置
```
~/.openclaw/
├── openclaw.json          # 主配置（含敏感信息，不备份）
├── workspace-boss/        # 管家配置
├── workspace-coder/       # 技术搭子配置
├── workspace-finance/     # 理财搭子配置
├── workspace-news/        # 新闻搭子配置
├── workspace-office/      # 办公搭子配置
├── workspace-planner/     # 方案搭子配置
├── workspace-reader/      # 读书搭子配置
```
