## 何时需要同步

当以下核心配置更新时，必须同步到整个团队：

| 配置类型 | 示例 | 影响范围 |
|:---|:---|:---|
| **发送规则** | 飞书图片/文件发送路径变更 | 所有助手 |
| **工作流程** | 视频学习流程、Markdown转PDF流程更新 | 相关助手 |
| **安全规范** | 配置修改规则、重启规范变更 | 所有助手 |
| **助手分工** | 职责调整、新增助手 | 所有助手 |
| **API配置** | Tavily API Key、邮件配置变更 | 相关助手 |

## 同步步骤

```markdown
1. 【更新】先更新我的配置文件（测试验证）
   └─ 确保新规则我自己先能用

2. 【归档】如有旧内容，标记为过期或归档
   └─ memory/xxx.md → 标记【已过期】
   └─ 指向新的规范位置

3. 【共享】更新 `shared-memory/` 对应文件
   └─ 工作流程 → shared-memory/workflows/
   └─ 发送规则 → shared-memory/feishu-send-rules.md
   └─ 通用规范 → shared-memory/xxx.md

4. 【通知】告知各助手配置变更（飞书消息）
   使用统一通知模板

5. 【确认】确认各助手已更新（抽查或全部确认）
   └─ 关键配置：必须确认
   └─ 一般配置：抽查即可
```

## 常见问题

### Q: 小改动也需要全团队同步吗？
A: 判断标准：
- **影响多个助手** → 必须同步
- **只影响我自己** → 无需同步
- **安全相关** → 必须同步（无论影响范围）

### Q: 助手没有按时更新怎么办？
A: 
1. 再次提醒（飞书消息）
2. 延迟截止时间
3. 必要时直接帮其更新
4. 记录到 feedback/ 作为改进点

### Q: 如何确保同步不遗漏？
A: 使用 checklist：
```markdown
- [ ] 已更新我的配置
- [ ] 已归档旧内容
- [ ] 已更新 shared-memory/
- [ ] 已发送通知
- [ ] 已确认各助手更新
```
