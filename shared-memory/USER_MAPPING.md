## 📋 标准工作流程

### 【通用铁律】只要涉及主动给组长发消息，必须遵循！

```
步骤1: 从自己的记录中找到组长的 ID
        ├─ 当前有会话 → 读取 inbound_meta.sender_id
        └─ 无当前会话 → 从历史会话记录中查找
                    ↓
步骤2: 使用获取到的 ID 发送消息
                    ↓
步骤3: 消息成功送达组长
```

---

## ✅ 适用场景

- 组长要求发测试消息
- 任务完成主动汇报
- 异常/错误主动通知
- 定时任务提醒
- 助手之间互相提醒时需转发给组长
- **任何需要主动联系组长的情况**

---

## ❌ 禁止行为

- 从其他助手/总管的消息中复制 ID
- 使用固定的/硬编码的 ID
- 假设所有助手的 ID 都一样
- 不查找历史记录就放弃发送

---

## 💡 关键认知

**飞书多机器人机制:**
- 同一个组长，在不同机器人下有不同的 `open_id`
- 方案搭子看到的组长ID ≠ 办公搭子看到的组长ID
- 必须使用**自己会话中**的ID才能成功发送

**本映射表用途:**
- ✅ 故障排查时核对
- ✅ 验证消息来源
- ❌ **不作为消息发送的依据**
