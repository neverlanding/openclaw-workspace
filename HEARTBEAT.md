# HEARTBEAT.md - 心跳速查

**每30分钟触发** | **核心**: 必检+轮换 → 量化判断 → 精准汇报

---

## 🎯 铁律与责任

```
❌ 禁止：只回 HEARTBEAT_OK
✅ 必须：必检项每次执行 → 轮换项选1项 → 量化判断 → 汇报或记录

漏检=失职：未检导致系统宕机>30min、任务连续失败未发现、助手异常>6h未汇报
```

---

## 🔍 检查清单

### ⭐ 必检项（每次必须，<2min）
- [ ] **系统服务**: `lsof -i:18789,9175,18793 | grep LISTEN`
- [ ] **定时任务**: `grep -i error /tmp/openclaw/*.log | tail -3`
- [ ] **任务追踪**: `cat shared-memory/task-tracker.md | grep "待确认\|进行中"`（检查未完成任务）

### 🔄 轮换项（选1项，确保每类每天1次）
- **上午**: 邮件紧急/日历2h内/待办逾期
- **下午**: 助手活跃状态、超时任务
- **深夜**: 日志归档、清理/tmp/

---

## 🧠 量化判断标准

### 🔴 立即汇报（满足任一）
| 场景 | 标准 |
|:---|:---|
| 服务宕机 | 无法访问 >5分钟 |
| 任务失败 | 同一错误连续2次 |
| 助手离线 | 完全无响应 >4小时 |
| 关键逾期 | 待办逾期 >24小时 |
| **任务追踪** | **任务截止未完成 或 30分钟无确认** |

### 🟡 解决后汇报
- 服务重启恢复、助手响应慢但可联系、非关键逾期<24h

### 🟢 不汇报
- 瞬时波动(<2min)、已知周期问题、已安排处理

**口诀**: 宕机必报、重复必报、超4h必报；恢复可缓、单次可记

## 🚨 汇报与记录

**立即汇报格式（🔴）**:
```
🚨 紧急: [服务/任务/助手]异常 | 时间: HH:MM
量化: [宕机X分钟/连续X次/离线Xh] | 影响: [范围] | 需决策: [事项]
```

**记录标准**:
- **正常**: `### 心跳 HH:MM | 服务:正常 | 任务:正常 | 轮换:邮件 | 行动:无`
- **异常**: `### 心跳 HH:MM 异常 | 检查:[项] | 量化:[标准] | 级别:[🔴🟡] | 汇报:[时间]`

**追踪**: `memory/heartbeat-state.json`
