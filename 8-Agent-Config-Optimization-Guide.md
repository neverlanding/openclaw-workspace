## 🎯 核心优化原则

### 1. 精简 ≠ 删减价值
- 删除冗余：解释性文字、重复描述、过度举例
- 保留核心：关键动作、判断标准、底线红线
- 表格化：用表格替代长段落描述
- 引用分离：详细内容放专门文件，主文件只留速查

### 2. 职责清晰，避免重复
每个配置文件有且只有一个核心职责：
- **IDENTITY.md**: 身份标签（我是谁）
- **SOUL.md**: 人格核心（我是什么样的人）
- **AGENTS.md**: 工作规范（我做什么、怎么做）
- **TOOLS.md**: 技能清单（我用什么工具）
- **HEARTBEAT.md**: 心跳任务（我定期执行什么）
- **MEMORY.md**: 长期记忆（我记住什么）

### 3. 版本统一，便于维护
- 所有核心配置文件使用统一版本格式：`版本: v1.0 | 更新: YYYY-MM-DD`
- 重大更新升级版本号（v1.0 → v1.1 → v1.2）
- 小修改只更新日期

### 4. 主动性强，可执行
- 每个配置必须能指导实际行动
- 量化标准（如：>5分钟、连续2次、>4小时）
- 自检清单，确保不遗漏

### 二、关键决策记录

#### 决策1: 文件发送规则变更
- **旧规则**: 使用 `media/browser/` 目录发送飞书文件
- **新规则**: 默认直接发送，失败时复制到 `workspace-{name}/tmp/` 再试
- **影响范围**: 所有8个助手
- **生效时间**: 2026-03-19

#### 决策2: AGENTS.md定位明确
- **定位**: 搭子总管 = 8-Agent团队总指挥（不是普通执行者）
- **核心职责**: 任务分配、进度跟踪、配置管理、质量控制、异常监控
- **关键权限**: 最高指挥调度权（但调度前必须告知组长确认）

#### 决策3: 主动性铁律确立
- **核心**: "主动发现问题、主动汇报进展、主动检查异常"
- **禁止**: "等组长问了才汇报"
- **执行标准**: 量化判断（🔴立即汇报/🟡解决后报/🟢不汇报）

#### 决策4: 客观性原则
- **要求**: 真实汇报、基于事实、不自我偏袒
- **信息来源**: 官方文档 > 社区经验 > 历史记录 > 逻辑推理 > 谨慎测试
- **底线**: 不确定就说"不确定"，宁可承认不知道，也不编造

#### 决策5: 配置文件维护准则
- **原则**: 精简≠删减价值，保留核心≠冗长
- **判断标准**: 核心价值、可执行性、必要性、长度成本
- **目标行数**: IDENTITY(20-60)、SOUL(100-200)、AGENTS(100-250)、TOOLS(50-100)、HEARTBEAT(50-100)
