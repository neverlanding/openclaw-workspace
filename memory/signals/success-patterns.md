# 成功模式记录

## 模式1: 子Agent并行任务处理

**描述**: 将独立任务分配给多个子Agent并行执行，提高效率

**触发条件**:
- 多个独立任务需要同时处理
- 任务耗时较长（>1分钟）
- 用户希望继续当前对话

**执行步骤**:
1. 分析任务是否可并行
2. 询问用户是否并行执行
3. 启动子Agent处理各任务
4. 主会话保持响应
5. 收集子Agent结果并汇总

**成功要素**:
- 任务之间无依赖关系
- 明确子Agent职责边界
- 及时同步进度信息

**首次成功**: 2026-02-25（Token统计+图片分析并行任务）

## 模式3: 三层记忆系统架构

**描述**: 将记忆按热度分层管理（P0热/P1温/P2冷）

**架构**:
- P0: MEMORY.md + 最近7天日志 + SOUL.md
- P1: lessons/ + projects/ + decisions/ + feedback/ + verified/ + signals/

**成功要素**:
