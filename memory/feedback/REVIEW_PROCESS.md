# 反馈回顾流程

## 目的
定期回顾用户反馈，识别改进机会，持续提升系统质量。

## 回顾频率

### 每周回顾（轻量级）
- **时间**: 每周五下午
- **时长**: 15-30分钟
- **内容**:
  - 查看本周所有反馈记录
  - 识别重复出现的问题
  - 记录快速修复项

### 每月回顾（深度分析）
- **时间**: 每月最后一个工作日
- **时长**: 1-2小时
- **内容**:
  - 分析反馈趋势
  - 评估改进措施效果
  - 更新优先级列表
  - 制定下月改进计划

### 季度回顾（战略层面）
- **时间**: 每季度末
- **时长**: 半天
- **内容**:
  - 系统性问题根因分析
  - 架构/流程改进建议
  - 长期规划调整

## 回顾流程

### 1. 收集阶段
```bash
# 查看本周反馈
ls -la ~/.openclaw/workspace/memory/feedback/*.md

# 统计反馈数量
grep -l "反馈来源" ~/.openclaw/workspace/memory/feedback/*.md | wc -l
```

### 2. 分类阶段
将反馈按以下维度分类：
- **类型**: 正面反馈 / 改进建议 / 问题报告
- **系统**: 记忆系统 / 子Agent / 工具技能 / 其他
- **优先级**: P0紧急 / P1重要 / P2一般

### 3. 分析阶段
- 识别高频问题
- 评估用户满意度趋势
- 关联相关系统变更

### 4. 行动阶段
- 创建待办事项
- 分配责任人
- 设定完成时间
- 更新相关文档

## 回顾检查清单

- [ ] 所有反馈记录已阅读
- [ ] 反馈已分类整理
- [ ] 趋势分析已完成
- [ ] 行动计划已制定
- [ ] 相关文档已更新
- [ ] 下次回顾时间已安排

## 输出物

1. **回顾摘要**: 记录关键发现和决策
2. **改进行动清单**: 具体的待办事项
3. **经验总结**: 值得记录的经验教训

## 相关文件

- 反馈模板: `memory/feedback/TEMPLATE.md`
- 反馈记录: `memory/feedback/YYYY-MM-DD.md`
- 经验教训: `memory/lessons/`
