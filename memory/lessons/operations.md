### 端口冲突处理原则（2026-03-03）
**教训来源**: Star-Office-UI部署过程
**核心原则**: 遇到端口冲突时，应检测可用端口并切换，而不是强制停止现有进程

**正确做法**:
```bash
# 1. 检测端口是否被占用
lsof -i:端口号

# 2. 如果被占用，寻找可用端口
for port in 18791 18792 18793 18794; do
  if ! lsof -i:$port > /dev/null 2>&1; then
    echo "可用端口: $port"
    break
  fi
done

# 3. 使用新端口启动服务
# 修改配置文件或启动参数
```

**错误做法**（避免）:
```bash
# ❌ 不要直接杀掉现有进程
kill -9 $(lsof -ti:端口号)
```

**原因**:
- 现有进程可能是用户其他重要服务
- 强制终止可能导致数据丢失
- 优先选择和平共存而非强制抢占
