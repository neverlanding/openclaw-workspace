## 何时使用子Agent

| 场景 | 说明 | 示例 |
|:---|:---|:---|
| **长时间任务** | >5分钟的任务 | 视频转录、大量数据处理 |
| **并行化子任务** | 可分解的独立任务 | 同时分析多个文件 |
| **隔离环境操作** | 需要干净环境的操作 | 测试新技能、实验性操作 |
| **需重试的任务** | 可能出错、需自动重试 | 网络请求、API调用 |
| **复杂任务分解** | 多步骤复杂任务 | 视频学习全流程 |

## 子Agent文件发送规则（2026-03-19 更新）

### 核心原则

```
默认直接发送 → 失败时复制到 workspace 的 tmp/ 再试
```

### 详细步骤

| 步骤 | 操作 | 说明 |
|:---|:---|:---|
| 1 | 默认直接发送 | 使用 `media` 参数直接发送文件 |
| 2 | 失败时处理 | 复制到 `workspace-{name}/tmp/` |
| 3 | 重试发送 | 从 tmp/ 目录重新发送 |

### 各助手备用路径

| 助手 | 备用路径 |
|:---|:---|
| 方案搭子 | `~/.openclaw/workspace-planner/tmp/` |
| 办公搭子 | `~/.openclaw/workspace-office/tmp/` |
| 公众号搭子 | `~/.openclaw/workspace-writer/tmp/` |
| 新闻搭子 | `~/.openclaw/workspace-news/tmp/` |
| 读书搭子 | `~/.openclaw/workspace-reader/tmp/` |
| 理财搭子 | `~/.openclaw/workspace-finance/tmp/` |
| 代码搭子 | `~/.openclaw/workspace-coder/tmp/` |

### 操作示例

```bash
# 1. 直接尝试发送
message send --media /path/to/file.pdf

# 2. 如果失败，复制到 tmp/
cp /path/to/file.pdf ~/.openclaw/workspace-planner/tmp/

# 3. 从 tmp/ 重试发送
message send --media ~/.openclaw/workspace-planner/tmp/file.pdf
```

## 子Agent同步机制（必须执行）

子Agent完成任务后**必须**执行以下操作：

### 1. 写入 memory/YYYY-MM-DD.md

```markdown
### [子Agent名称] 任务完成

**时间**: YYYY-MM-DD HH:MM
**来源**: [主任务名称]
**状态**: ✅ 完成

**完成内容**:
- [具体的产出1]
- [具体的产出2]

**关键发现**:
- [重要的发现或决策]

**后续事项**:
- [ ] 需要主会话处理的事项1
- [ ] 需要主会话处理的事项2

**文件变更**:
- `path/to/file1` - 变更说明
- `path/to/file2` - 变更说明
```

### 2. 更新 memory/appointments.md（如有待办）

```markdown
- [ ] [待办事项] - [截止时间] - [来源任务]
```

### 3. 汇报给搭子总管

**格式**：
```
📋 任务完成报告
═══════════════════════════════════════
任务: [任务名称]
状态: [完成/部分完成/失败]
耗时: [X分钟]
───────────────────────────────────────
产出:
• [产出项1]
• [产出项2]

重要发现:
• [发现1]

需要跟进:
• [跟进项1]
═══════════════════════════════════════
```

## 子Agent禁止行为

- ❌ **禁止**：不经过搭子总管直接联系组长（除非紧急）
- ❌ **禁止**：修改 MEMORY.md（应由总管决定）
- ❌ **禁止**：执行破坏性操作而不记录
- ❌ **禁止**：忽略错误而不报告
- ❌ **禁止**：任务完成后不汇报

## 常见问题

### Q: 子Agent超时怎么办？
A: 
1. 设置足够超时（任务预估 × 1.5）
2. 超时后检查子Agent状态
3. 必要时终止并重试
4. 记录超时原因到日志

### Q: 子Agent报错怎么办？
A:
1. 收集错误信息
2. 判断是否可重试
3. 重试或终止任务
4. 完整记录错误到日志

### Q: 如何确保子Agent不遗漏同步？
A: 在任务描述中明确要求：
```
任务完成后必须：
1. 写入 memory/YYYY-MM-DD.md
2. 汇报给搭子总管
```
