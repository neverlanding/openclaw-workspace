## 系统健康检查详解

### 定时任务状态检查

**目的**: 发现 Cron 任务失败、报错

**检查命令**:
```bash
# 查看最近的错误日志
grep -i "error\|fail\|exception" /tmp/openclaw/*.log | tail -20

# 查看特定日期日志
tail -100 /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log | grep -i error
```

**判断标准**:
- 单条错误 + 已恢复 → 🟢 记录，无需汇报
- 同一任务重复失败 → 🔴 立即汇报
- 未知错误类型 → 🟡 先查看详情，再判断

**常见错误及处理**:
| 错误 | 处理 |
|:---|:---|
| "Unexpected event order" | 重启 Gateway，观察是否恢复 |
| "Connection refused" | 检查服务状态，重启 |
| "Timeout" | 检查网络/依赖服务 |

### 系统服务状态检查

**目的**: 确保核心服务运行正常

**检查命令**:
```bash
# 检查端口监听
lsof -i:18789,9175,18793 | grep LISTEN

# 或检查进程
ps aux | grep -E "openclaw|clawalytics|office-ui"

# 检查服务响应
curl -s http://localhost:18789/health || echo "Gateway down"
```

**服务清单**:
| 服务 | 端口 | 检查方式 |
|:---|:---:|:---|
| Gateway | 18789 | lsof / curl |
| 虾壳面板 | 9175 | lsof / 浏览器访问 |
| 办公室视图 | 18793 | lsof / 浏览器访问 |

**故障处理**:
1. 检查端口占用: `lsof -i:端口号`
2. 查看日志: `tail -f /tmp/openclaw/*.log`
3. 尝试重启: `openclaw gateway restart`
4. 无法恢复 → 🔴 立即汇报

### 助手状态巡检

**目的**: 发现助手异常、任务阻塞

**检查内容**:
1. 各助手最后活跃时间
2. 长时间未完成的任务（>2小时）
3. 待办事项逾期情况

**检查方式**:
```bash
# 查看各助手会话列表
ls -lt ~/.openclaw/workspace-*/memory/2026-03-*.md | head -10

# 检查待办
cat ~/.openclaw/workspace-boss/memory/appointments.md
```

**判断标准**:
| 情况 | 行动 |
|:---|:---|
| 助手1-2小时无响应 | 先发消息询问 |
| 助手>4小时无响应 | 🔴 汇报组长 |
| 任务超时但助手正常 | 🟡 询问进展 |
| 待办逾期<1天 | 🟡 汇总后汇报 |
| 待办逾期>1天 | 🔴 立即汇报 |

## 信息监控详解

### 邮件检查

**目的**: 发现紧急邮件

**检查方式**: 通过邮件客户端或 web 界面

**汇报标准**:
- 标题含"紧急"/" ASAP" → 🔴 立即汇报
- 来自组长的未读 → 🔴 立即汇报
- 普通邮件 → 🟢 记录，无需汇报

---

### 日历检查

**目的**: 提前提醒即将发生的事件

**检查方式**: 通过日历应用或 API

**提醒标准**:
- 事件<2小时 → ✅ 提醒组长
- 事件 2-24小时 → 🟢 记录
- 事件>24小时 → 不提醒

---

### 待办事项检查

**目的**: 跟进关键待办

**检查方式**: 读取 `memory/appointments.md`

**判断标准**:
| 状态 | 行动 |
|:---|:---|
| 今日到期 | 🔴 提醒组长 |
| 明日到期 | 🟡 记录，明日优先检查 |
| 已逾期<1天 | 🟡 汇总汇报 |
| 已逾期>1天 | 🔴 立即汇报 |

## 深夜维护任务详解

### 日志确保

**目的**: 确保每日日志存在，便于追踪

**操作**:
```bash
# 检查当日日志
file="$HOME/.openclaw/workspace-boss/memory/$(date +%Y-%m-%d).md"
if [ ! -f "$file" ]; then
  echo "# $(date +%Y-%m-%d) 工作日志" > "$file"
fi
```

### 清理临时文件

**目的**: 释放磁盘空间

**操作**:
```bash
# 删除7天+的临时文件
find /tmp -type f -mtime +7 -delete 2>/dev/null

# 清理各 workspace 的 old 临时文件
find ~/.openclaw/workspace-*/tmp -type f -mtime +7 -delete 2>/dev/null
```

## 预测性维护详解

### 趋势分析方法

**磁盘空间趋势**:
```bash
# 每周记录磁盘使用
df -h >> ~/.openclaw/memory/stats/disk-usage.log

# 检查增长趋势
tail -10 ~/.openclaw/memory/stats/disk-usage.log
```

**任务失败频率**:
```bash
# 统计本周错误数
grep -c "ERROR" /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log
```

### 容量预警阈值

| 指标 | 黄色预警 | 红色预警 |
|:---|:---:|:---:|
| 磁盘空间 | >70% | >85% |
| 内存使用 | >70% | >90% |
| 日志大小 | >100MB | >500MB |

## 常见问题 FAQ

**Q: 检查项太多，每次做不完？**
A: 只选1-2项，轮换执行。关键是坚持，不是一次做完。

**Q: 不确定该不该汇报？**
A: 问自己：如果我是组长，我想知道这件事吗？想 → 汇报。

**Q: 每天都报同样的异常怎么办？**
A: 说明是系统性问题，需要彻底解决，不是简单汇报。
