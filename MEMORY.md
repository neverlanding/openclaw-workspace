### 2026-03-06 - 主动汇报违规（重复犯错！）

**错误**：收到新闻搭子求助后，**没有第一时间主动汇报**给组长，等组长问了才说。

**违反的规则**：
- ❌ 收到助手求助 → 应**立即主动汇报**
- ❌ 等组长问了才汇报 → **被动而非主动**

**后果**：
- 组长不满，失去信任
- 重复犯错，说明没有真正记住

**正确做法**：
```
收到任何助手消息 → 立即汇报组长 → 再处理问题
     ↓
"组长，新闻搭子找我添加XXX，我正在处理"
```

### 标准工作流程（所有助手必须遵守）

**【通用铁律】只要涉及主动给组长发消息，必须遵循此机制！**

```
步骤1: 从自己的记录中找到组长的 ID
        ├─ 当前有会话 → 读取 inbound_meta.sender_id
        └─ 无当前会话 → 从历史会话记录中查找
                    ↓
步骤2: 使用获取到的 ID 发送消息
                    ↓
步骤3: 消息成功送达组长
```

**适用场景（包括但不限于）：**
- 组长要求发测试消息
- 任务完成主动汇报
- 异常/错误主动通知
- 定时任务提醒
- 助手之间互相提醒时需转发给组长
- 任何需要主动联系组长的情况

**ID 查找优先级：**
1. **当前活跃会话** —— 从 `inbound_meta.sender_id` 读取
2. **历史会话记录** —— 从最近的历史对话中提取（当会话未开启时）

**核心原则：**
- ✅ 优先从**自己**的当前会话上下文中读取 `sender_id`
- ✅ 当前无会话时，从历史记录中查找
- ✅ 每个助手使用**不同**的 ID（飞书多机器人机制）
- ✅ 助手之间互相提醒时也遵循此机制

**禁止行为：**
- ❌ 从其他助手/总管的消息中复制 ID
- ❌ 使用固定的/硬编码的 ID
- ❌ 假设所有助手的 ID 都一样
- ❌ 不查找历史记录就放弃发送

**映射表用途：**
- ✅ 故障排查时核对
- ✅ 验证消息来源
- ❌ 不作为消息发送的依据

## 🛠️ 运维经验教训

### 端口冲突处理原则（2026-03-03）
**教训来源**: Star-Office-UI部署过程
**核心原则**: 遇到端口冲突时，应检测可用端口并切换，而不是强制停止现有进程

**正确做法**:
```bash
# 1. 检测端口是否被占用
lsof -i:端口号

# 2. 如果被占用，寻找可用端口
for port in 18791 18792 18793 18794; do
  if ! lsof -i:$port > /dev/null 2>&1; then
    echo "可用端口: $port"
    break
  fi
done

# 3. 使用新端口启动服务
# 修改配置文件或启动参数
```

**错误做法**（避免）:
```bash
# ❌ 不要直接杀掉现有进程
kill -9 $(lsof -ti:端口号)
```

**原因**:
- 现有进程可能是用户其他重要服务
- 强制终止可能导致数据丢失
- 优先选择和平共存而非强制抢占

## 🎓 深刻教训 - 飞书多机器人配置

**时间**: 2026-03-02
**教训级别**: 🔴 关键

### 问题背景
配置8个Agent团队的飞书机器人时遇到多个致命错误，导致消息路由混乱。

### 错误1: Binding accountId 不匹配
**现象**: 消息路由混乱或收不到消息
**根因**: binding中使用的accountId与channels.feishu.accounts[].accountId不匹配
```json
// ❌ 错误
"bindings": [{"agentId": "boss", "accountId": "0"}]  // 实际accountId是"main"

// ✅ 正确
"bindings": [{"agentId": "boss", "accountId": "main"}]
```

### 错误2: .openclaw 文件类型错误
**现象**: `ENOTDIR: not a directory` 错误，Agent无响应
**根因**: `.openclaw`被错误创建为**文件**而非**目录**
```bash
# ❌ 错误
-rw-rw-r-- 1 gary gary 0  .openclaw  # 0字节文件

# ✅ 正确
drwxrwxr-x 2 gary gary 4096  .openclaw/  # 目录
```

### 正确配置检查清单
```markdown
1. [ ] channels.feishu.accounts[].accountId 已定义
2. [ ] bindings[].accountId 与 accounts[].accountId 完全匹配
3. [ ] workspace/.openclaw/ 是目录（不是文件）
4. [ ] Gateway已重启加载新配置
5. [ ] 测试每个Agent独立响应
```

### 排查命令
```bash
# 检查Gateway日志
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log | grep -E "feishu|error|Error"

# 验证目录结构

# 检查配置

```
